Lesson 6: Getting Comfortable With Common Tasks

String Formatting

We'll often have data that we'd like to print to standard output, write to a file, or embed in a website.

What not to do

When programmers start learning Python, one of the first things they learn is the string concatenation operator, +.

>>> "choco" + "late"
'chocolate'

But it doesn't take long to realize that this method can be very error prone and inefficient, not to mention difficult to read!

For example, we need to add spaces manually between words:

>>> "Hello" + "World!"
'HelloWorld'
>>> "Hello" + " World!"
'Hello World!'

Not only does that look awkward, but it can be really easy to forget. It gets even worse when our data isn't a string type.

>>> person = {'name': "Lucas", 'age': 24}
>>> "I am " + person['name'] + " and I am " + str(person['age']) + "!"
'I am Lucas and I am 24!'

Ew, don't do that.

There's a better way!

Now, that's not to say string concatenation isn't useful and should always be avoided. Rather, it should be avoided when our purpose is formatting. Instead, we should use either the string formatting operator, or preferably, the format() method.

>>> "I am {} and I am {}!".format(person['name'], person['age'])
'I am Lucas and I am 24!'

We simply define a template string and then provide the data to replace.

Each pair of braces {} indicates a placeholder for some data. Placeholders will be replaced in the order that the arguments are passed.

Repeated and rearranged arguments

If we need to repeat the same piece of data multiple times or reorder the arguments in the string we can use indices inside the braces:

>>> tag = 'h1'
>>> content = "Hello World!"
>>> "<{0}>{1}</{0}>".format(tag, content)
'<h1>Hello World!</h1>'

Named arguments

Sometimes indices make it difficult to see exactly what's going on. We can provide keyword arguments to the format() method to make things clearer.

>>> "<{tag}>{content}</{tag}>".format(tag='h1', content="Hello World!")
'<h1>Hello World!</h1>'

We can combine this with argument unpacking to simply our example from before.

>>> "I am {name} and I am {age}!".format(**person)
'<h1>Hello World!</h1>'

Property access

If you have a dictionary (or any data type that maps a key to a value) you can access its properties directly inside the template string.

>>> my_dict = {'value': "potato"}
>>> "The value is: {[value]}".format(my_dict)

Combining this with indexing leads to some very compact code.

>>> "I am {0[name]} and I am {0[age]}!".format(person)
'I am Lucas and I am 24!'

This also works with objects created from custom classes! Assume we have a Person object with both name and age properties.

class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

Just use the property access operator, ., instead of dictionary indexing.

>>> person = Person("Lucas", 24)
>>> "I am {0.name} and I am {0.age}!".format(person)
'I am Lucas and I am 24!'

A few more cool features

Padding with 0

>>> "{month} {day:02}, {year}".format(month='August', day=5, year=1930)
'August 05, 1930'

Adding commas to numbers

>>> "{:,}".format(1000000000)
'1,000,000,000'

Decimal rounding

>>> "Euler's number is approximately {e:.2f}".format(e=2.718281828459045)
"Euler's number is approximately 2.72"

Organizing Code

Modules

In Python, files are called modules. As you might expect, modules can get pretty long and complex. We will often break our logic out into separate modules in order to keep things a little more organized. This process is sometimes referred to as modularizing our code. Python modules end with a .py extension.

Packages

If modules are like files, then packages are like directories (i.e. folders). Python packages give us a nice way to group similar modules under a common namespace. Inside every package is a module called __init__.py, which tells Python that this isn't just a directory, it's a package.

Importing modules and packages

If the module you want to import is in the same directory as your script (or working directory in a shell), then all you need to do is add the following line to top of your file.

import example_module

Packages can also be imported this way.

import example_package

Or, if you're only interested in specific parts of the package, there is a special syntax.

from example_package import example_module_1, example_module_2

You may come across the following line in someone's code:

from example_package import *

In this case, import * means import everything from the module or package. Avoid this whenever possible since it can lead to lots of unused code that may dramatically increase file sizes. Import statements are like copying and pasting one file into another.

Builtin modules and packages

The Python standard library comes with an extensive list of packages and modules with builtin logic to perform common tasks. Let's take a look at a couple of simple packages to get started.

Pickle

While working on a project we might create an object that we know we'll to reuse later on down the line. It would be extremely useful if we could just save the data from our Python objects and load it up again whenever we need it. This process is called pickling!

Note: we MUST open pickle files with the binary, b, mode because they are not plaintext files!

Saving Python objects to a file

To write objects to a file, we use the dump() method.

my_object = MyObject() 
with open('my_object_file.pkl', 'wb') as file:
    pickle.dump(my_object, file)

Retrieving Python objects from a file

To read objects from a file, we use the load() method. Note the we must open the file in rb mode because these are not plaintext files.

with open('my_object_file.pkl', 'rb') as file:
    my_object = pickle.load(file)

JSON

JavaScript Object Notation (JSON) is a file format for representing structured data. It is one of the most common formats for information exchange in use today. Much like the pickle module, the json module is used to serialize (and deserialize) data. In addition to the load() and dump() functions, json exposes two other methods called loads() and dumps(). These methods should not be read as plurals, they stand for "load string" and "dump string" respectively.

NOTE: JSON is a plaintext format, so binary mode is NOT necessary

Reading from a file

with open('data_file.json', 'r') as read_file:
    data = json.load(read_file)

Reading from a string

json_string = "{"person": {"name": "Lucas", "age": 24}}"
data = json.loads(json_string)

Writing to a file

with open('data_file.json', 'w') as write_file:
    json.dump(data, write_file)

Writing to a string

data = {'person': {'name': "Lucas", 'age': 24}}
json_string = json.dumps(data)

Dates and Times

Manipulating dates and times is a very common task, and luckily Python has a builtin package called datetime for just that purpose!

There are four main classes we need to be aware of: date, time, datetime, timedelta

Dates

The datetime.date class represents our notion of calendar days. Each instance of the class has year, month, and day attributes.

>>> bday = datetime.date(1930, 8, 5)
>>> bday.year
1993
>>> bday.month
8
>>> bday.day
5

Times

The datetime.time class represents a time of the day. Each instance of has hour, minute, second, and microsecond attributes.

>>> t = datetime.time(14, 36, 12, 342570)
>>> t.hour
14
>>> t.minute
36
>>> t.second
12
>>> t.microsecond
342570

Datetimes

The datetime.datetime class represents the combination of a datetime.datetime object and a datetime.time object. Each instance of has both date and time related attributes.

Timedeltas

To understand these objects, consider the expression

timedelta = datetime - datetime

So datetime.timedelta objects are simply the difference of two date- or time-like objects.

We can therefore rearrange the expression to see that timedeltas can be added to (or subtracted from) datetimes to get previous or future datetimes.

timedelta + datetime = datetime
